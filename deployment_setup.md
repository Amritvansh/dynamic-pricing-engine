# 🚀 Deployment Environment Setup

> **Project:** Dynamic Pricing Engine
> **Platform Stack:** MongoDB Atlas (DB) + Render (Node.js) + Vercel (React)

---

## Phase 1: Database (MongoDB Atlas)

1. **Create Cluster:** Go to [mongodb.com/atlas](https://www.mongodb.com/atlas) and create a Free Tier (M0) cluster.
2. **Database Access:** 
   - Create a database user.
   - Save the username and autogenerated password securely.
3. **Network Access:**
   - Go to "Network Access" in the sidebar.
   - Add IP Address: `0.0.0.0/0` (Allow Access from Anywhere). *This is crucial so Render can connect to your DB.*
4. **Get Connection String:**
   - Click "Connect" -> "Connect your application".
   - Copy the connection string. Replace `<password>` with your real password.

---

## Phase 2: Backend Deployment (Render.com)

1. **Create Web Service:**
   - Log into [Render.com](https://render.com).
   - Click **New** -> **Web Service**.
   - Connect your GitHub repository.
2. **Configure Build:**
   - **Root Directory:** `backend` (Important! Tell Render where the Node app lives).
   - **Environment:** `Node`
   - **Build Command:** `npm install`
   - **Start Command:** `node server.js`
3. **Environment Variables:**
   - Go to the "Environment" tab in Render and add:
     - `MONGO_URL` = Your MongoDB Atlas connection string.
     - `PORT` = `5000`
     - `GEMINI_API_KEY` = Your Google AI Studio API key.
4. **Deploy:** Click "Save and Deploy". Wait for the green "Live" status. Copy the deployed URL (e.g., `https://dynamic-pricing-api.onrender.com`).

---

## Phase 3: Frontend Deployment (Vercel)

1. **Create Project:**
   - Log into [Vercel.com](https://vercel.com).
   - Click **Add New** -> **Project**.
   - Import your GitHub repository.
2. **Configure Build:**
   - **Framework Preset:** `Vite`
   - **Root Directory:** `frontend` (Click edit and select the frontend folder).
3. **Environment Variables:**
   - Add the following variable:
     - `VITE_API_URL` = `https://dynamic-pricing-api.onrender.com` (Your Render backend URL).
4. **Deploy:** Click "Deploy". Vercel will automatically run `npm run build` inside the frontend folder and host the `dist/` output.

---

## Phase 4: CORS Configuration Verification

Before presenting the project, ensure the backend allows requests from the frontend.

In `backend/server.js`, ensure CORS is configured either to accept all origins or specifically the Vercel URL:

```javascript
const cors = require('cors');

// For development/college project purposes:
app.use(cors({
    origin: '*' 
}));

// STRICT PRODUCTION SECURITY (Optional):
// app.use(cors({
//    origin: 'https://your-frontend.vercel.app'
// }));
```

## 🔒 Security Checklist Before Submission

- [ ] Ensure `.env` is listed in your `.gitignore` and NEVER pushed to GitHub.
- [ ] Do not hardcode your `GEMINI_API_KEY` directly into `pricingEngine.js`.
- [ ] If your MongoDB password contains special characters like `@` or `#`, ensure it is URL-encoded in the connection string.
